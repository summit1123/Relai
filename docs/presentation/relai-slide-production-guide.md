# RelAi 슬라이드 제작 가이드

## 사용 목적
- `RelAi_pitch_deck_draft.pptx`를 빠르게 수정할 때 기준이 되는 제작 가이드
- 슬라이드별로 무엇을 크게 보여주고 무엇을 말로 풀지 구분하기 위한 문서

## 전체 원칙
- 한 슬라이드에는 한 메시지만 남긴다
- 글보다 구조가 먼저 읽혀야 한다
- 제품은 "챗봇"처럼 보이면 안 되고, 운영 보드/의사결정 시스템처럼 보여야 한다
- 흰 배경, 검정 텍스트, 연한 블루 포인트만 사용한다

## 메인 슬라이드 구성

### 1. 표지
- 크게: RelAi
- 작게: 한 줄 가치 제안
- 우측: 컨셉보드 이미지

### 2. 문제 정의
- 가운데: 기획자 -> 개발자 -> QA -> 승인권자 흐름
- 주변 태그: 변경, 재작업, 승인 지연, 회의 반복

### 3. 왜 지금인가
- 상단: AI 활용 확산 수치
- 하단: 개인 생산성 상승 vs 조직 정렬 정체 대비

### 4. 인사이트
- 큰 문장 한 줄
- 배경은 여백을 넓게 두고 임팩트 중심

### 5. RelAi 정의
- 좌: 입력
- 중: AI 구조화
- 우: 출력

### 6. 핵심 플로우
- 초안 등록 -> 역할별 해석 -> 합의 잠금 -> 영향 맵 -> 메모리 -> AI 회의
- 화살표형 흐름을 유지

### 7. 시나리오
- 실제 변경 요청 한 줄
- 좌: 영향 항목 요약
- 우: 영향 맵 스크린샷

### 8. 제품 화면
- 큰 화면 1개
- 보조 화면 2개
- 캡션은 짧게만

### 9. 차별성
- 표 비교
- 기존 도구 vs RelAi

### 10. 실현 가능성 및 사업성
- 3분할 카드
- 실현 가능성 / 초기 고객 / 수익 모델

### 11. 클로징
- 큰 문장 1개
- 부연 문장 1개

## Appendix 운용 방식
- 질문이 나오면 바로 넘길 수 있게 제목을 명확히 둔다
- 수치 질문 -> Appendix A
- 구조 질문 -> Appendix B, C
- 기존 도구 질문 -> Appendix D
- 시장 질문 -> Appendix E
- 리스크 질문 -> Appendix F
- 짧은 즉답 -> Appendix G

## 최종 수정 포인트
- 발표 직전에는 문장 길이를 한 번 더 줄인다
- 이미지가 작게 많기보다, 핵심 화면이 크게 보이게 조정한다
- 차별성 슬라이드 문구는 가장 공격적으로 다듬는다
