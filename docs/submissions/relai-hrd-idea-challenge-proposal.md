# Relai 제출용 기획서 초안

## 참가신청서 1페이지 요약

### 아이디어명

Relai

### 참가 부문

재직자 / AI 업무혁신 부문

### 아이디어 한 줄 요약

Relai는 요구사항 변경, 팀 채팅, 회의 기록, AI 산출물을 하나의 프로젝트 메모리로 통합하고, 이를 결정·질문·승인·영향·액션 단위로 구조화해 기획-개발-품질 간 재작업과 승인 지연을 줄이는 AI 업무혁신 시스템입니다.

### 제안 배경

저는 AI 소프트웨어 개발자로 일하며, 기획이 나오면 개발을 진행하고, 개발 중간에 기획이 수정되며, 품질 검수 과정에서 다시 개발 범위가 바뀌는 상황을 반복적으로 경험했습니다. 이 문제는 특정 직군의 역량 부족이라기보다, AI로 개인의 작업 속도는 빨라지고 있는데 요구사항 변경을 합의하고 전파하는 방식은 여전히 메신저, 회의, 문서 중심에 머물러 있기 때문에 발생한다고 보았습니다.

특히 기획자, 개발자, 품질 담당자는 모두 AI를 활용하더라도 서로 다른 목적과 방식으로 사용합니다. 기획자는 AI로 사용자 시나리오와 정책 문구를 빠르게 구체화하고, 개발자는 AI로 코드와 구현 방식을 빠르게 설계하며, 품질 담당자는 AI로 테스트 관점과 예외 케이스를 확장합니다. 하지만 이 결과물이 하나의 합의 구조로 다시 정렬되지 않으면 같은 요구사항도 서로 다른 방향으로 빠르게 확장될 수 있습니다.

### 해결 아이디어

Relai는 요구사항 초안, 변경 요청, 팀 채팅, 회의 기록을 AI가 분석해 결정·질문·승인·영향·액션·메모리로 분류합니다. 변경이 발생하면 이전 대화와 회의 맥락까지 참고해 변경 영향 맵과 합의 잠금 보드를 생성하고, 채팅만으로 합의가 어려운 경우 AI가 회의 안건과 참석자를 제안합니다. 회의가 끝난 뒤에는 회의 기록을 다시 프로젝트 메모리로 저장해 다음 변경 검토에 활용합니다.

### 기대효과

- 요구사항 변경 검토 시간 단축
- 기획-개발-품질 간 해석 차이 감소
- 회의 후 결정 누락 감소
- 승인 지연과 재작업 비용 감소
- 조직 내 업무 지식의 프로젝트 메모리화

# 5페이지 기획서 본문

## 1페이지. 문제정의: AI 시대에도 요구사항 변경은 여전히 팀을 느리게 만든다

### 본문

AI 도구의 확산으로 개인의 업무 속도는 빠르게 높아지고 있습니다. Microsoft Work Trend Index 2024에 따르면 지식근로자의 75%가 업무에서 AI를 사용하고 있으며, AI 사용자 중 78%는 회사가 제공한 도구가 아니라 개인적으로 선택한 AI 도구를 업무에 가져와 사용하고 있습니다. Gallup 조사에서도 미국 직장인의 업무상 AI 사용은 2023년 21%에서 2025년 40%로 증가했습니다.

하지만 AI 활용이 늘어난다고 해서 팀의 합의 속도까지 자동으로 높아지는 것은 아닙니다. 오히려 각자가 AI를 다른 방식으로 사용하면서 새로운 협업 간극이 생길 수 있습니다. 기획자는 AI로 사용자 시나리오와 정책 문구를 구체화하고, 개발자는 AI로 구현 방식과 코드 구조를 설계하며, 품질 담당자는 AI로 테스트 케이스와 예외 상황을 확장합니다. 같은 요구사항을 보고도 각자가 다른 AI 산출물을 기준으로 판단하면, 기획 의도, 구현 범위, 품질 기준이 서로 다른 방향으로 빠르게 커질 수 있습니다.

이 문제는 실제 수치로도 확인됩니다. Atlassian의 2025 Developer Experience 조사에서는 개발자 68%가 AI로 주 10시간 이상을 절약한다고 답했지만, 동시에 개발자 50%는 정보 탐색, 도구 전환, 협업 문제 등 조직 비효율로 주 10시간 이상을 잃고 있었습니다. 또한 개발자가 실제 코딩에 쓰는 시간은 약 16%에 불과하며, 나머지 84%는 요구사항 확인, 정보 탐색, 회의, 승인, 협업 등 코드 주변 업무에 쓰입니다.

의사소통 문제도 재작업으로 이어집니다. Atlassian/Loom의 2026년 조사에서는 근로자 80%가 글로 된 메시지를 오해해 불필요하거나 다시 해야 하는 일을 경험했다고 응답했으며, 미국 기준 연간 생산성 손실은 약 2,030억 달러로 추정되었습니다. PMI 2021 조사에서도 프로젝트 실패 원인으로 요구사항 수집 부실 40%, 프로젝트 목표 변경 38%, 커뮤니케이션 부실 28%가 제시되었습니다.

따라서 이 문제는 단순히 기획자와 개발자가 말을 더 잘해야 하는 문제가 아닙니다. AI로 작업 속도가 빨라진 시대에, 요구사항 변경과 의사결정 흐름을 팀 단위로 다시 정렬하는 AX 기반 업무 운영 구조가 필요합니다.

## 2페이지. 서비스 개요: Relai는 요구사항 변경을 팀의 실행 단위로 정렬한다

### 본문

Relai는 요구사항 변경, 팀 채팅, 회의 기록, AI 산출물을 하나의 프로젝트 메모리로 통합하고, 이를 결정·질문·승인·영향·액션 단위로 구조화하는 AI 협업 운영 시스템입니다.

기존 협업 도구는 문서, 티켓, 채팅, 회의록을 각각 저장하는 데 강점이 있습니다. 그러나 변경이 발생했을 때 어떤 결정이 다시 열렸는지, 어떤 화면과 API가 영향을 받는지, 누가 승인해야 하는지, 이전 회의에서 어떤 조건이 합의되었는지를 한 번에 보여주지는 못합니다. Relai는 바로 이 지점을 해결합니다.

Relai의 핵심 사용자는 PM/PO, 서비스 기획자, 개발 리드, 품질 담당자, 승인권자입니다. 이들은 같은 프로젝트 안에 있지만 서로 다른 관점으로 요구사항을 바라봅니다. Relai는 하나의 변경 요청을 각 역할의 실행 언어로 다시 정리하고, 변경 영향과 합의 상태를 함께 보여줍니다.

### 핵심 기능

| 기능 | 설명 |
| --- | --- |
| 요구사항 초안 등록 | 기획 초안, 정책 변경, 기능 요청을 프로젝트 단위로 등록합니다. |
| AI 관점 변환 | 같은 요구사항을 기획자 관점, 개발자 관점, 품질 관점으로 재구성합니다. |
| 합의 잠금 보드 | 잠긴 결정, 열린 질문, 승인 필요, 착수 가능 조건을 분리합니다. |
| 변경 영향 맵 | 변경이 화면, API, DB, 테스트, 일정, 담당자에게 미치는 영향을 표시합니다. |
| 프로젝트 메모리 | 이전 채팅, 회의, 결정, 기술 제약을 메모리 카드로 저장합니다. |
| AI 회의 개설 | 채팅만으로 합의가 어려운 경우 회의 필요성을 감지하고 안건을 제안합니다. |
| 회의 기록 컨텍스트화 | 회의 녹음과 회의록을 다시 결정·질문·승인·액션으로 구조화합니다. |

### Relai의 차별점

Relai는 단순한 AI 채팅앱이나 회의록 요약 도구가 아닙니다. 핵심은 대화 내용을 예쁘게 요약하는 것이 아니라, 요구사항 변경을 팀이 실행할 수 있는 단위로 바꾸는 것입니다. 즉, 문장을 저장하는 것이 아니라 문장을 결정, 질문, 승인, 영향, 액션, 근거, 메모리로 나눠 관리합니다.

## 3페이지. 사용자 시나리오: 변경 요청이 들어온 순간, 팀은 이렇게 정렬된다

### 본문

예를 들어, 기획자가 “관리자 승인 후 결제를 확정하는 정책을 이번 릴리즈에 포함하고 싶다”고 요청했다고 가정합니다. 일반적인 협업 방식에서는 이 변경이 단순한 정책 문구 수정처럼 보일 수 있습니다. 하지만 실제 개발 현장에서는 이 변경이 결제 완료 화면, 결제 확정 API, 데이터베이스 필드, 승인 전후 테스트 케이스, QA 일정, 담당자 배정, 승인권자 판단까지 연결될 수 있습니다.

Relai는 이 변경 요청을 단순 텍스트로 두지 않고 변경 영향 맵으로 전환합니다. AI는 이전 회의와 채팅 기록, 기존 결정 메모리, 요구사항 초안을 함께 참고해 무엇이 이미 잠긴 결정인지, 무엇이 다시 열린 항목인지, 누가 결정해야 하는지를 구조화합니다.

### 시나리오 흐름

1. 기획자가 변경 요청을 등록합니다.
2. AI가 기존 요구사항과 프로젝트 메모리를 함께 조회합니다.
3. 변경이 기존 합의와 충돌하는지 확인합니다.
4. 화면, API, DB, 테스트, 일정, 담당자, 승인권자 영향 범위를 도출합니다.
5. 합의 잠금 보드에서 잠긴 결정과 다시 열린 질문을 분리합니다.
6. 채팅으로 해결이 어렵다고 판단되면 AI가 회의 필요성을 제안합니다.
7. 회의 후 결정과 액션은 다시 프로젝트 메모리로 저장됩니다.

## 4페이지. 기술 구조: 프로젝트 메모리가 팀의 합의를 계속 성장시킨다

### 본문

Relai의 핵심 기술 구조는 프로젝트 메모리입니다. 요구사항 문서, 팀 채팅, 회의 기록, 녹음, AI 산출물은 모두 프로젝트 컨텍스트로 유입됩니다. AI는 이를 단순 저장하지 않고 결정, 질문, 승인, 영향, 액션, 근거 링크, 메모리 카드로 구조화합니다.

이 구조가 중요한 이유는 요구사항 변경이 항상 현재 문장 하나만으로 판단되지 않기 때문입니다. 과거 회의에서 “이 기능은 2차 릴리즈로 미룬다”고 합의했을 수도 있고, 개발 리드가 “결제 API 변경 시 QA 기간이 2일 더 필요하다”고 말했을 수도 있으며, 승인권자가 “운영 리스크 때문에 수동 승인 옵션은 유지해야 한다”고 남겼을 수도 있습니다. 이런 맥락이 사라지면 팀은 같은 논의를 반복하게 됩니다.

Relai는 새 변경 요청이 들어왔을 때 이전 대화와 회의 기록을 함께 참고합니다. 그리고 채팅만으로 합의가 어렵거나 충돌 신호가 커지면 AI가 회의 필요성을 판단하고, 회의 제목, 참석자, 핵심 안건, 사전 확인 자료, 결정해야 할 항목을 제안합니다.

회의가 끝나면 녹음 또는 회의록은 다시 AI에 의해 구조화됩니다. 새 결정, 열린 질문, 승인 필요 항목, 액션 아이템, 메모리 저장 항목이 생성되고, 이는 다음 변경 검토의 근거가 됩니다. 이 과정을 통해 Relai는 일회성 분석 도구가 아니라 팀의 맥락을 축적하는 협업 운영 시스템이 됩니다.

### 기술 흐름

| 단계 | 입력/처리 | 결과 |
| --- | --- | --- |
| 1 | 요구사항, 변경 요청, 채팅, 회의 기록 수집 | 프로젝트 컨텍스트 생성 |
| 2 | AI가 결정·질문·승인·영향·액션으로 분류 | 운영 단위 구조화 |
| 3 | 중요한 맥락을 메모리 카드로 저장 | 프로젝트 메모리 구축 |
| 4 | 새 변경 요청 발생 시 이전 메모리 검색 | 맥락 기반 영향 분석 |
| 5 | 충돌 또는 미결정 감지 | 회의 안건 제안 |

## 5페이지. 기대효과와 사업성: 재작업을 줄이는 AI 업무혁신 레이어

### 본문

Relai의 기대효과는 문서 작성 시간을 줄이는 데 그치지 않습니다. 핵심은 요구사항 변경으로 인해 발생하는 재작업, 승인 지연, 회의 반복, 결정 누락을 줄이는 것입니다.

Atlassian 조사처럼 개발자가 AI로 시간을 절약하더라도 조직 비효율로 같은 수준의 시간을 다시 잃는다면, AI 도입 효과는 제한될 수밖에 없습니다. Relai는 개발 자체보다 개발 주변에서 발생하는 요구사항 정렬, 정보 탐색, 회의, 승인, 품질 피드백의 병목을 줄이는 데 초점을 둡니다.

특히 재직자 AI 업무혁신 부문에서 Relai는 실무성이 분명합니다. 실제 기업 현장에서는 기획, 개발, 품질, 승인자가 동시에 움직이며, 변경 요청은 거의 항상 여러 업무 영역에 영향을 줍니다. Relai는 이 과정을 AI로 구조화해 팀이 더 빠르고 정확하게 합의하도록 돕습니다.

### 기대효과

| 기대효과 | 설명 |
| --- | --- |
| 변경 검토 시간 단축 | 변경 요청이 들어왔을 때 영향 범위를 자동 구조화합니다. |
| 재작업 감소 | 기존 결정과 충돌하는 변경을 조기에 발견합니다. |
| 승인 지연 감소 | 승인권자가 판단해야 할 항목과 리스크를 명확히 봅니다. |
| 회의 효율 개선 | AI가 필요한 회의만 안건 중심으로 제안합니다. |
| 결정 누락 감소 | 회의 후 결정과 액션을 다시 보드와 메모리에 반영합니다. |
| 조직 지식 축적 | 반복되는 정책, 기술 제약, 품질 기준을 프로젝트 메모리로 남깁니다. |

### 사업화 방향

| BM | 대상 | 설명 |
| --- | --- | --- |
| 팀 단위 SaaS 구독 | 제품팀, 개발팀, PM 조직 | 월 구독 형태로 프로젝트별 협업 공간 제공 |
| 프로젝트 단위 과금 | 외주/고객사 협업 프로젝트 | 프로젝트 기간 동안 변경 관리와 의사결정 기록 제공 |
| 협업 패키지 | 기획사, 개발사, QA 조직 | 요구사항 변경, 품질 피드백, 승인 브리핑 기능 묶음 제공 |
| 엔터프라이즈 플랜 | 중견·대기업 | 보안, 권한 관리, 내부 시스템 연동, 감사 로그 제공 |

### 확장 가능성

- Jira, Notion, Slack, Teams 연동
- 회의 녹음 자동 구조화
- 코드 저장소 기반 변경 영향 분석
- QA 테스트 케이스 자동 연결
- 조직별 업무 메모리 구축
- 외주 계약 변경 관리 기능 확장

### 마무리 문장

Relai는 AI로 더 많은 문서를 만드는 도구가 아니라, AI 시대에 더 빠르게 발생하는 요구사항 변경과 해석 차이를 팀 단위의 합의 구조로 바꾸는 업무혁신 시스템입니다. 개인의 AI 활용을 넘어, 기획-개발-품질-승인자가 함께 일하는 방식을 AX 관점에서 재설계하는 것이 Relai의 목표입니다.

### 페이지 핵심 메시지

Relai는 AI 코딩 도구가 해결하지 못하는 84%의 개발 주변 업무, 즉 요구사항 정렬과 협업 운영의 병목을 줄이는 제품입니다.
