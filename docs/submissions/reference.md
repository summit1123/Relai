# ReLine 참고자료

## 서비스명

ReLine

포항형 MICE 운영 변경 정렬 플랫폼

## 서비스 개요

ReLine은 포항에서 열리는 MICE 행사에서 초안, 변경 요청, 영향 범위, 역할별 대응을 한 화면에서 정리하는 운영 정렬 서비스입니다. 핵심은 문서를 더 많이 만드는 것이 아니라, 변경이 들어왔을 때 누가 무엇을 먼저 확인해야 하는지와 어떤 파트너가 영향을 받는지를 즉시 읽히게 만드는 데 있습니다.

## 한눈에 보기

| 항목 | 요약 |
| --- | --- |
| 핵심 문제 | 변경이 생길 때 기준안, 영향 범위, 승인 항목이 흩어집니다 |
| 핵심 해법 | 초안, 변경 요청, 영향 보드, 브리핑을 한 흐름으로 잠급니다 |
| 1차 사용자 | 주최기관, 운영사, 지역 협력 파트너 |
| 기대 변화 | 반복 설명 감소, 승인 속도 향상, 현장 혼선 감소 |

## 왜 지금 필요한가

포항은 행사장 하나만 준비하면 끝나는 도시가 아닙니다. 산업 행사, 해양관광, 지역 체류 프로그램이 함께 묶일 수 있는 구조이기 때문에, 일정이나 인원 변화가 생기면 숙박, 식음, 교통, 안전, 현장 인력이 동시에 흔들립니다. 이 구조에서는 단순 공지보다 운영 정렬이 더 중요합니다.

| 한국관광공사 보고서 | MICE 전체 참가자 863만 명, 공급비용 약 6,097억 원, 평균 총 체류일 3.46일 | 운영 정렬은 지역 소비와 체류 품질에 영향을 주는 실무 과제입니다 |

## 문제를 어떻게 정의하는가

포항형 MICE의 문제는 "아이디어가 부족하다"가 아니라, 변경이 생겼을 때 그 영향을 운영 단위로 다시 잠글 구조가 부족하다는 데 있습니다.

| 변경 상황 | 실제로 다시 맞춰야 하는 항목 |
| --- | --- |
| 참가 인원 증가 | 객실 수, 식음 수량, 체크인 인력, 좌석 |
| 세션 시간 지연 | 셔틀, 입장 동선, 후속 프로그램, 안내 메시지 |
| 장소 변경 또는 야외 프로그램 취소 | 대체 공간, 안전요원, 현장 표지물, 파트너 전달 |

## 데이터 포인트

![ReLine MICE 규모](/Users/gimdonghyeon/Desktop/Relai/docs/submissions/assets/reline-mice-scale.svg)

이 수치는 ReLine이 단순한 협업 툴이 아니라 운영 리스크를 줄이는 도구여야 한다는 근거입니다. 참가자 체류와 지역 소비가 연결된 산업에서는 변경이 늦게 정렬될수록 손실이 커집니다.

## 도입 판단 포인트

| 판단 항목 | 현재 운영 방식 | ReLine 도입 시 변화 |
| --- | --- | --- |
| 기준안 관리 | 최신안이 메신저, 메일, 문서에 흩어집니다 | 행사 초안을 기준안으로 잠그고 이후 변경을 연결합니다 |
| 영향 파악 | 담당자 경험에 따라 다시 확인합니다 | 숙박, 식음, 교통, 안전, 안내 영향을 한 번에 확인합니다 |
| 승인 판단 | 지금 무엇이 미정인지 다시 물어봐야 합니다 | 승인 필요 항목과 다음 액션을 짧게 압축합니다 |
| 파트너 전달 | 같은 내용을 여러 번 재전달합니다 | 파트너별 필요한 조정만 분리해 전달합니다 |

이 판단 포인트는 포항의 지역 구조와 특히 잘 맞습니다. 포항은 산업 기반 행사와 지역 체류형 프로그램을 함께 엮기 쉬워서, 변경이 생겼을 때 행사장 내부 조정만으로 끝나지 않습니다. 실무에서는 "행사 하나"처럼 보이지만 실제로는 행사장, 숙박, 식음, 교통, 안전, 안내 인력이 같이 움직이는 작은 운영망에 가깝습니다.

## 포항 적용 장면

산업 포럼과 지역 체류 프로그램이 함께 묶인 1박 2일 행사를 기준으로 보면 ReLine의 쓰임이 더 분명합니다. 행사 3일 전에 인원이 늘고 세션 시간이 밀리며 야외 프로그램이 실내로 이동하면, 운영팀은 숙박, 식음, 셔틀, 안전, 안내를 동시에 다시 맞춰야 합니다. ReLine은 이때 변경을 한 건으로 잠그고, 각 파트너가 다시 확인해야 할 항목만 분리해 전달하는 역할을 맡습니다.

| 현장 상황 | ReLine이 먼저 잠그는 것 |
| --- | --- |
| 참가 인원 증가 | 객실 추가, 식음 증량, 체크인 인력 증원 |
| 세션 시간 지연 | 셔틀 시각, 안내 문구, 후속 일정 영향 |
| 야외 프로그램 실내 전환 | 대체 공간, 안전요원, 동선 표지물, 참가자 공지 |

## 초기 MVP 범위

ReLine의 초기 범위는 과하게 크지 않아야 합니다. 먼저 행사 초안, 변경 요청, 영향 범위, 승인 브리핑이라는 핵심 루프가 실제로 시간을 줄이는지만 검증하면 됩니다.

| MVP 기능 | 포함 범위 | 우선 검증할 질문 |
| --- | --- | --- |
| 초안 등록 | 일정, 장소, 인원, 파트너, 목표 시점 입력 | 기준안이 한 곳에 잠기는가 |
| 역할별 정렬 | 주최기관용, 운영사용, 파트너용 요약 | 같은 초안을 각 역할이 바로 이해하는가 |
| 변경 요청 패널 | 제목, 요청자, 긴급도, 이유, 반영 희망 시점 기록 | 변경이 구두 전달이 아니라 추적 가능한가 |
| 영향 범위 보드 | 숙박, 식음, 셔틀, 안전, 안내, 인력 영향 표시 | 누가 먼저 움직여야 하는지가 빨리 보이는가 |
| 의사결정 브리핑 | 승인 필요 항목, 미정 항목, 다음 액션 | 승인자의 판단 시간이 줄어드는가 |

## 기대 효과

ReLine의 효과는 세 가지로 요약할 수 있습니다. 첫째, 운영사는 같은 변경을 여러 파트너에게 반복 설명하는 시간을 줄일 수 있습니다. 둘째, 승인권자는 현재 무엇이 잠겼고 무엇이 미정인지 더 빨리 파악할 수 있습니다. 셋째, 지역 파트너는 자신과 관련된 변경만 정확하게 전달받을 수 있어 대응 누락이 줄어듭니다.

이 기대 효과는 포항이라는 지역 조건과도 맞닿아 있습니다. 포항은 산업행사, 해양관광, 체류형 프로그램이 함께 묶일 수 있는 도시라서 운영 변경이 숙박과 교통, 현장 안전까지 넓게 퍼집니다. 따라서 ReLine의 가치는 협업 편의가 아니라 포항형 행사 운영의 손실을 줄이는 데 있습니다.

## 왜 포항형 제안인가

전국 어디서나 쓸 수 있는 일반 협업툴을 들고 와서는 포항 공모전에서 설득력이 약합니다. ReLine은 포항의 현재 방향을 전제로 잡습니다. 포항은 POEX와 영일만관광특구, 해양관광 인프라, 산업도시 기반이 겹치면서 앞으로 더 많은 복합형 행사 운영을 요구받을 가능성이 큽니다. 즉, 지금 필요한 것은 단순 홍보 도구보다 "변경이 들어오면 지역 파트너 전체를 다시 정렬하는 운영 도구"입니다.

이 점에서 ReLine은 포항형 MICE 운영의 빈칸을 겨냥합니다. 행사를 유치한 뒤 어떤 순간에 손실이 커지는지, 어떤 변경이 반복적으로 숙박과 교통과 안전까지 번지는지, 누가 승인 지점에서 막히는지를 데이터로 남길 수 있기 때문입니다. 장기적으로는 이 데이터가 포항 행사 운영의 표준을 만드는 자산이 될 수 있습니다.
